How to Create a Rule

Before you can migrate or replicate data, a replication rule needs to be created.

  1. From the rules section of the dashboard, go to the create a rule page.
  2. Define the rule you wish to create:
    • Give the rule a unique name (i.e. one you haven't used before).
    • Provide the path on the source zone you want to replicate data from.
    • If different, also give the path on target zone to replicate the data to.
  3. Files or directories can be excluded from replication using glob patterns. The following exclusions are present by default: 
    /**/.fusion
/**/.fusion/**
/**/.Trash
/**/.Trash/**
/**/.hive-staging**
    These exclude Fusion’s housekeeping files, trash directories and Hive staging directories. You can add any other exclusions required.
